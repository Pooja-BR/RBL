Exstream License Server Installation Readme

This ReadMe file contains basic information that you need to install the FlexNet Publisher License Server Manager and the Exstream Licensing Vendor Daemon that are included in the License_Server_Install_<version>.zip.

System requirements

Make sure you meet the following system requirements for installing the FlexNet Publisher License Server Manager:

Platform

Operating System

Microsoft Windows (32-bit and 64-bit)
  • Windows 10
  • Windows Server 2019
  • Windows Server 2016

Tip: As a best practice, OpenText recommends installing license servers on a server-based OS.

Installing the License Server

To enable floating licenses, you must complete the following steps:

  1. Extract the contents of the License_Server_Install_<version>.zip file.

  2. Install the FlexNet Publisher License Server Manager using the lmadmin-<version>.exe file.

    Important: Before you install the Flexnet Publisher License Server Manager, make sure that you have installed the Java Runtime Environment (JRE) version 1.6 or later.

    You must install the Flexnet Publisher License Server Manager before installing the Exstream Licensing Vendor Daemon. If you have already installed the current or a newer version of FlexNet Publisher License Server Manager, then you can skip this step.

  3. Install the Exstream Licensing Vendor Daemon using the OpenText_Exstream_Licensing_setup.exe file.

Installing the FlexNet Publisher License Server Manager

When you are installing the FlexNet Publisher License Server Manager, accept the default settings unless otherwise specified in the following steps:

  1. Right-click the lmadmin-<version>.exe file and select Run as administrator.

    Follow the directions in the FlexNet Publisher License Server Installer wizard to install the FlexNet Publisher License Server Manager (lmadmin).

    Important: You must run the installer as an administrator to correctly install the FlexNet Publisher License Server Manager.

  2. On the Service Configuration screen, select the Run as service check box.

  3. After you have completed the installation, to enable the return of a borrowed license before the borrow return date, complete the following steps:

    1. At the Windows command prompt, make sure that you are in the FlexNet Publisher License Server Manager directory.

    2. At the command prompt, enter lmadmin –adminonly no and press Enter.

    3. Restart the lmadmin service.

After you have completed the FlexNet Publisher License Server installation steps, install the Exstream Licensing Vendor Daemon.

Installing the Exstream Licensing Vendor Daemon

  1. Run the OpenText_Exstream_Licensing_setup.exe that was included in the License_Server_Install_<version>.zip file.

  2. On the OpenText Exstream Licensing <version> Installation window, click Next.

  3. On the Select License File screen, do one of the following:

    • If you have a license file, make sure that the I have a license file check box is selected and click Browse to select your license file.

    • If you do not have a license file, clear the I have a license file check box. You can import your license file after you have completed the installation using the FlexNet License Administrator.

  4. Click Next.

  5. On the Ready to Install screen, click Install.

  6. On the Installation Complete screen, click Finish.

    During installation, the Exstream Licensing Vendor Daemon installer copies your license file to the same location as the Exstream Licensing Vendor Daemon (for example, C:\Program Files (x86)\FlexNet Publisher License Server Manager\exstream).

Note: If the Exstream license file that you have selected is part of a three-server setup, and you have existing vendor daemons (other than the Exstream daemon) currently installed, the FlexNet Publisher License Server Manager cannot successfully import the Exstream license file. Before you import the Exstream license file, you must uninstall any existing vendor daemons (other than the Exstream daemon).

After you have installed the FlexNet Publisher License Server and the Exstream Licensing Vendor Daemon, you can start the FlexNet License Administrator by opening a web browser and going to http://localhost:8090. If you changed the local port during the FlexNet Publisher License Server installation, replace 8090 with the new local port that you specified during installation.

You can use the FlexNet License Administrator to modify the ports assigned to the server and vendor daemon. Keep in mind that if you do not specify a port number for the vendor daemon, a port number will be assigned by the operating system during run time. If the port that the operating system assigns is behind a firewall, communication with the licensing vendor daemon might be unsuccessful.

For more information about advanced license server administration options, see the FlexNet Publisher Administration guide included in the ZIP or installed in C:\Program Files (x86)\FlexNet Publisher License Server Manager\exstream. Not all options in the guide are enabled or applicable for use with Exstream.

In addition to the standard license server log files stored in the C:\ProgramData\FLEXlm\lmadmin\logs directory, the Exstream vendor daemon generates log files that record license transactions on the license server. A new log file (exstream_usage_<yyyy>_<mm>.log) is generated for each month, and is stored in the C:\ProgramData\FNP\FLEXlm directory.

The transaction log file provides a useful option for administrators to troubleshoot problems or to determine if additional seats might be needed if users are frequently unable to check out a license. Keep in mind that if a license is checked out when a month rolls over and a new log file is created, the check-in time for that license will not be recorded in the log file for either the current month or the previous month.

To effectively manage storage on your server, OpenText recommends that you periodically review and delete or archive older log files.

Uninstalling the License Server

To uninstall the license server, you must first uninstall the Exstream Licensing Vendor Daemon, and then uninstall the FlexNet Publisher License Server.

Uninstalling the Exstream Licensing Vendor Daemon

Keep in mind that after you uninstall the Exstream Licensing Vendor Daemon, the exstream folder is removed from C:\Program Files (x86)\FlexNet Publisher License Server Manager, but it is not removed from the FlexNet Administrator list of vendor daemons.

To uninstall the Exstream Licensing Vendor Daemon:

  1. Use the Windows uninstall tool to uninstall OpenText Exstream Licensing.

  2. On the OpenText Exstream Licensing window, select the Remove from this computer check box.

  3. Click Next.

  4. On the Remove Licenses screen, do one of the following:

    • If you want to remove all Exstream license files during the uninstall, make sure that the I want to leave all OpenText Exstream license files installed on this computer check box is cleared.

    • If you do not want to remove all Exstream license files, select the I want to leave all OpenText Exstream license files installed on this computer check box.

  5. Click Next.

  6. On the Ready to Remove screen, click Remove.

  7. On the Removal Complete screen, click Finish.

    The Exstream Licensing Vendor Daemon is removed from your computer.

Uninstalling the FlexNet Publisher License Server Manager

Before uninstalling the FlexNet Publisher License Server Manager, make sure you are not using other FlexNet-enabled applications.

To uninstall the FlexNet Publisher License Server Manager, use the Windows uninstall tool and follow the directions in the wizard.
